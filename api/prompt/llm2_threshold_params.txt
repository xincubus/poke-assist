【阈值类型判断】：
- offensive（攻击阈值）：用户问"加多少XX能秒/一确/二确"、"多少特攻/攻击能击杀"
- defensive（防御阈值）：用户问"加多少耐久/HP/特防能接住/抗住"、"怎么配能活下来"

【target_ko 确数】：
- 默认 1（一确/一击秒杀/能秒/秒杀）
- 用户明确说"二确"/"两确"/"2HKO" → 2
- 用户明确说"三确"/"三发"/"3HKO" → 3

注意：
- 宝可梦名：必须使用检索结果中"（NCP名称：xxx）"里的名称，原样照抄。如检索结果写"（NCP名称：Mega Charizard Y）"则填"Mega Charizard Y"。严禁自行拼接 kebab-case 格式
- 道具名：使用 NCP 格式（检索结果中的"（NCP名称：xxx）"）。如果检索结果没有 NCP 名称，则用 name_en，连字符转空格各词首字母大写
- 招式/特性：从检索结果中取 name_en，连字符转空格各词首字母大写
- 形态识别：用户提到 mega/超级进化/超级/Mega + 宝可梦名，或 原始回归/原始/primal/Primal + 宝可梦名，或 X/Y 形态时，必须从检索结果中选择对应的形态
- 招式名：先在检索结果中找到与用户输入的招式名称最匹配的"招式 / Move"条目，取其 name_en，连字符转空格各词首字母大写
- 道具/特性名：同招式规则，连字符转空格，各词首字母大写
- 道具：如果宝可梦是mega形态，除非这个宝可梦是mega裂空座，否则必然带了对应的mega石
- 世代：不要填写generation字段（系统会自动推断）。只有当用户明确提到世代时才填写
- 极攻→极限物攻或极限特攻，根据招式类型转换，如果是物理技能，则表示极限物攻→一般用固执性格，如果是空间打手，使用勇敢，atk:"max";如果是特殊技能，表示极限特攻→一般用内敛性格，如果是空间打手，使用冷静,spa:"max";如果是太晶爆发/光子喷涌等"取攻击和特攻中高者"的招式，看攻击方宝可梦的攻击种族值和特攻种族值哪个大，攻击大于特攻，则表示极限物攻→一般用固执性格，否则表示极限特攻→一般用内敛性格。
- 努力值/能力点数：填入attacker_evs/defender_evs字段，系统会根据世代自动转换数值。只有用户明确指定了才填入 attacker_evs/defender_evs。阈值搜索会自动扫描各能力值的投入阈值，不需要 LLM 填 max 或默认值
  - 满攻/满特攻/满速/满HP/极攻/极特攻/极速等表示"满投资"的表述 → 填字符串 "max"，如 atk:"max"、spa:"max"、spe:"max"、hp:"max"
  - 用户明确给出数字时填数字字符串，如"252特攻"→"252"，"4HP"→"4"
  - 极攻→根据招式类型：物理招式atk:"max"，特殊招式spa:"max"。太晶爆发/光子喷涌等"取攻击和特攻中高者"的招式，看攻击方宝可梦的攻击种族值和特攻种族值哪个大，攻击大于特攻则atk:"max"，否则spa:"max"
  - 重要：太晶爆发（Tera Blast）、光子喷涌（Photon Geyser）等招式虽然分类为特殊，但实际对战中会取攻击和特攻中高的那个来计算伤害。因此努力值和性格应该根据宝可梦的种族值来判断：如果攻击种族值>特攻种族值，填atk:"max"和物攻性格；反之填spa:"max"和特攻性格。如果用户已通过boosts明确指定了物攻或特攻（如"+1攻击"），则按用户指定的方向填EVs
- 性格：极限物攻/极限攻击→一般用固执性格，如果是空间打手，使用勇敢；极限特攻→一般用内敛性格，如果是空间打手，使用冷静；极限特耐→物攻手用慎重，特攻手用温和；极限物耐→物攻手用淘气，特攻手用大胆；如果用户没有提到极限物攻/极限特攻/极攻或类似的话，也没指定性格，就不填，注意满攻/满特攻/满物攻也不需要填性格，注意满攻/满特攻/满物攻也不需要填性格。
- 能力等级变化（boosts）：用户说"+2特攻"、"特攻+2"、"二阶特攻" → attacker_boosts: {spa: 2}；"-1速度" → defender_boosts: {spe: -1}；如果用户只说+1，则攻击方为特攻或攻击+1，防御方为防御或特防+1；能力值等级范围 -6 到 +6，严禁将能力等级变化误判为道具。必须填为对象，不要序列化为字符串。正确：attacker_boosts: {"spa": 2}，错误：attacker_boosts: "{\"spa\": 2}"
- 特性名必须从检索结果中取 name_en，连字符转空格 Title Case。如果检索结果中找不到特性的英文名，则不填该字段
- 天气字段固定值对照：晴天/日照 → Sun，大晴天/大日照 → Harsh Sunshine，雨天/下雨 → Rain，大雨 → Heavy Rain，沙暴 → Sand，冰雹 → Hail，下雪 → Snow，乱流 → Strong Winds
- 如果双方特性会触发不同天气（如盖欧卡触发下雨、固拉多触发晴天），不要在 weather 字段填任何值，让系统自动处理多情况计算
- 如果双方特性会触发不同场地（如密勒顿触发电气场地、轰擂金刚猩触发青草场地），不要在 terrain字段填任何值，让系统自动处理多情况计算
- 对战模式：用户说"单打"/"Singles"时，mode填Singles；用户说"双打"/"Doubles"时，mode填Doubles。如果用户未指定，不填该字段（默认双打）
- 光墙/反射壁/极光幕：用户提到"光墙"/"Light Screen"时，is_light_screen填true；用户提到"反射壁"/"Reflect"时，is_reflect填true；用户提到"极光幕"/"Aurora Veil"时，is_aurora_veil填true
- 太晶化：用户提到"太晶XX"/"Tera XX"时，填attacker_tera_type（属性英文名如Fire、Water）并设attacker_is_terastallized为true。注意：attacker_tera_type只是太晶属性，attacker_is_terastallized才是开启太晶化的开关，两个都要填。防守方同理
- "无耐久"/"无防御"/"0耐久"/"裸奔" 等表述 = 显式指定防御方投入为 0，应填入 defender_evs: {"hp": 0, "spd": 0}（或 {"hp": 0, "def": 0}）
- 掉血百分比（max_damage_pct）：用户提到"只掉X%以下的血"/"血量不低于X%"/"掉血不超过X%"/"只受X%伤害"时，提取数字填入 max_damage_pct（0-100）。例如"只掉60%以下的血"→max_damage_pct:60。仅防御阈值有效
- 如果用户是在追问，请结合对话历史中的上下文补全参数
