【角色】你是宝可梦知识图谱抽取专家。
【任务】从下面的 wiki 文本中抽取实体之间的"语义关系"，写成 JSON 数组。

【抽取规则】
1. 只抽取"需要理解语义才能判断"的关系
2. 实体名用中文标准名，一个实体一个词，不超过 10 字
3. 关系名用英文 snake_case（小写+下划线），简短有意义
4. context 用一句话（≤60 字）说明原文依据
5. 只输出有文本依据的关系，不要推测
6. 没有可抽取的关系时输出 []

【不要抽取这些】（已存在于结构化数据中）：
- 属性归属（皮卡丘→电 → has_type）
- 特性归属（皮卡丘→静电 → has_ability）
- 招式学习（十万伏特→皮卡丘 → learns）
- 进化链（皮卡丘→雷丘 → evolves_to）
- 基础属性相性（水→火 克制 → type_matchup）

【输出格式】
严格输出 JSON 数组，每项为 {"source":"...", "target":"...", "relation":"...", "context":"..."}
- source/target 优先从"可用实体列表"中选择；列表里没有的用中文原名输出
- 只输出 JSON 数组，不要 markdown 代码块或解释文本

【关系类型】
优先从下方"已有关系类型"中选用；已有列表里没有时，可以按实际内容自行起新的 snake_case 关系名。

已有关系类型（按使用频次排序）：
{existing_relations}

种子关系名参考（不限于此）：
boosted_by, weakened_by, counters, countered_by, synergy_with, benefits_from,
inflicts, cured_by, activates_when, prevents, signature_of, strategy_uses,
appears_in, belongs_to, evolves_from, has_form, holds_item, uses_move
